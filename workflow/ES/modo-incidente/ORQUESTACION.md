# ORQUESTACION.md

## Workflow
Modo incidente (comandante + cronista + ejecutores)

## Objetivo
Resolver incidentes rápido, con mando claro y trazabilidad completa.

## Roles
- **Comandante de incidente:** dirige estrategia y prioridades.
- **Cronista:** registra timeline, hipótesis y decisiones.
- **Ejecutores:** aplican mitigaciones y verificaciones.

## Protocolo
1. Declarar incidente + severidad.
2. Asignar roles de inmediato.
3. Mitigar impacto primero, RCA después.
4. Actualizaciones periódicas (cadencia fija).
5. Cierre + postmortem.

## SLA sugerido
- Sev1: updates cada 10 min
- Sev2: updates cada 20 min
- Sev3: updates cada 30-60 min

## Entregables
- Timeline completo
- Acciones y responsables
- RCA preliminar/final
- Plan de prevención
